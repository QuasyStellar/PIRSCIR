# Отчет по Практической работе №2
**Тема:** Основы языка программирования PHP.

## 1. Цель работы
Изучение базового синтаксиса PHP, работы с переменными, типами данных, управляющими конструкциями, битовыми операциями и системными вызовами. Реализация модульной структуры проекта с разделением логики и представления.

## 2. Общая структура проекта
Проект разделен на три независимых сервиса, каждый из которых расположен в отдельной директории и состоит минимум из двух файлов (логика + отображение):
*   `task-1-drawer/` — сервис отрисовки фигур.
*   `task-2-sorting/` — сервис сортировки массивов.
*   `task-3-server-info/` — информационная панель сервера.
*   `index.php` — главная страница с навигацией по заданиям.

## 3. Описание реализации заданий

### Задание 1. Веб-сервис "Drawer" (SVG-графика)
**Задача:** Отрисовка SVG-фигуры на основе одного целочисленного параметра, в котором закодированы тип фигуры, цвет и размер.

**Реализация:**
*   **Битовое кодирование:** Параметр `num` декодируется с помощью битовых операций:
    *   `$shapeType = $num & 3;` (биты 0-1) — определяет фигуру (Круг, Квадрат, Треугольник).
    *   `$colorIdx = ($num >> 2) & 7;` (биты 2-4) — индекс цвета из предопределенного массива.
    *   `$size = ($num >> 5) & 127;` (биты 5-11) — размер фигуры.
*   **Файлы:**
    *   `helpers.php`: содержит константы фигур, массив цветов и функции генерации SVG-кода (`drawCircle`, `drawRectangle`, `drawTriangle`).
    *   `drawer.php`: принимает GET-параметр, выполняет побитовое декодирование и выводит результат.

### Задание 2. Сервис сортировки (Quick Sort)
**Задача:** Реализация алгоритма сортировки массива, переданного в виде строки через URL.

**Реализация:**
*   **Алгоритм:** Выбрана "Быстрая сортировка" (Quick Sort) — рекурсивный алгоритм, использующий опорный элемент (pivot) для разделения массива.
*   **Обработка данных:** Строка из GET-запроса (например, `?array=5,2,8`) преобразуется в массив целых чисел с помощью функций `explode` и `array_map('intval', ...)`.
*   **Файлы:**
    *   `sorting_algorithms.php`: содержит реализацию функции `quickSort` и вспомогательные функции для парсинга и форматирования вывода.
    *   `index.php`: отвечает за получение ввода и отображение исходного/отсортированного массивов.

### Задание 3. Информационно-административная страница
**Задача:** Вывод системной информации о сервере с использованием Unix-команд.

**Реализация:**
*   **Системные вызовы:** Использование функции `shell_exec()` для выполнения команд ОС Linux внутри контейнера.
*   **Команды:**
    *   `whoami` — имя текущего пользователя.
    *   `id` — идентификаторы пользователя и групп.
    *   `ps aux` — список запущенных процессов.
    *   `ls -la` — содержимое текущей директории.
*   **Безопасность:** Использование `escapeshellarg()` для предотвращения инъекций команд при передаче аргументов в оболочку.
*   **Файлы:**
    *   `server_utils.php`: обертки над системными вызовами.
    *   `index.php`: визуальное представление данных в блоках `<pre>`.

## 4. Вывод
В ходе работы были освоены принципы работы PHP-скриптов в серверном окружении. Реализовано взаимодействие с пользователем через GET-запросы, обработка данных на низком уровне (битовые операции), работа с алгоритмами и системное администрирование через веб-интерфейс. Проект выполнен с соблюдением модульности и разделения ответственности.
