# Отчет по Практической работе №1
**Тема:** Развертывание изолированной инфраструктуры разработки (LAMP-стек на базе Docker).

## 1. Цель работы
Создать изолированную, воспроизводимую среду серверного программного обеспечения, включающую веб-сервер (Apache), интерпретатор (PHP) и систему управления базами данных (MySQL), с использованием технологий контейнеризации Docker. Проверить работоспособность связки компонентов путем вывода данных из БД на веб-страницу.

## 2. Архитектура решения
Для выполнения задачи был выбран подход «Инфраструктура как код» (Infrastructure as Code) с использованием инструмента оркестровки **Docker Compose**. Архитектура состоит из двух взаимодействующих контейнеров:
1.  **СУБД (База данных):** Контейнер на базе официального образа `mysql:8.0`.
2.  **Веб-сервер + PHP:** Кастомный контейнер на базе образа `php:8.2-apache`, содержащий веб-сервер Apache и интерпретатор PHP с необходимыми расширениями.

## 3. Пошаговое описание реализации

### Шаг 1. Конфигурация базы данных (MySQL)
В файле `docker-compose.yml` был описан сервис `db`. Для обеспечения безопасности и удобства, параметры доступа (пароли, названия) были переданы через переменные окружения:
*   `MYSQL_DATABASE: appDB` — автоматическое создание базы данных при первом запуске.
*   `MYSQL_USER: user` и `MYSQL_PASSWORD: password` — создание пользователя приложения.

Для автоматического создания структуры таблиц и их заполнения (сидирования) был написан скрипт `init.sql`. Через механизм Docker Volumes этот скрипт пробрасывается в специальную директорию контейнера `/docker-entrypoint-initdb.d/`, благодаря чему он автоматически выполняется при инициализации базы. Скрипт:
1. Выдает права пользователю `user`.
2. Создает таблицу `users` (с полями `ID`, `name`, `surname`).
3. Добавляет 5 тестовых записей (Alex Rover, Bob Marley и др.), используя конструкцию `INSERT ... SELECT`, чтобы избежать дублирования данных при перезапусках.

### Шаг 2. Настройка веб-сервера и PHP
Для веб-приложения потребовалась сборка собственного образа, так как базовый образ PHP не включает в себя драйвер для работы с MySQL. 
Был написан `Dockerfile`:
```dockerfile
FROM php:8.2-apache
RUN docker-php-ext-install mysqli
EXPOSE 80
```
Он берет за основу Debian с установленным Apache и PHP 8.2, доустанавливает расширение `mysqli` (необходимое для подключения к БД) и декларирует порт 80.

В `docker-compose.yml` для сервиса `app`:
*   Настроен проброс порта хоста `8081` на порт `80` контейнера. Приложение будет доступно по адресу `http://localhost:8081`.
*   Настроено монтирование файлов исходного кода (`index.php`, `style.css`) внутрь контейнера в папку `/var/www/html/`. Это позволяет редактировать код на хост-машине и сразу видеть изменения в браузере без пересборки контейнера (hot-reload).
*   Установлена жесткая зависимость (`depends_on: condition: service_healthy`) от базы данных. Веб-сервер не запустится, пока MySQL не будет полностью готов принимать подключения, что исключает ошибку "Connection refused" при старте.

### Шаг 3. Разработка тестового приложения
Для верификации работы стека был написан скрипт `index.php`.
1.  **Подключение к БД:** Используется класс `mysqli`. В качестве хоста базы данных указывается имя сервиса из `docker-compose` — `"db"`. Docker имеет встроенный DNS-сервер, который автоматически разрешает имя сервиса во внутренний IP-адрес контейнера базы данных.
2.  **Вывод данных:** Выполняется SQL-запрос `SELECT * FROM users`, результаты в цикле форматируются и выводятся в виде HTML-таблицы. Для визуального оформления таблица стилизуется через подключенный файл `style.css`.
3.  **Информация о среде:** В конце страницы вызывается функция `phpinfo()`, которая генерирует подробный отчет о конфигурации PHP, установленных модулях (включая `mysqli`) и параметрах Apache.

## 4. Запуск и проверка
Запуск всей инфраструктуры осуществляется одной командой в терминале из директории проекта:
`docker-compose up -d` (флаг `-d` запускает контейнеры в фоновом режиме).

После успешной сборки образа и инициализации базы данных, при переходе в браузере по адресу `http://localhost:8081` отображается:
1. Заголовок "Таблица пользователей данного продукта".
2. HTML-таблица с данными (Имена и Фамилии), успешно извлеченными из MySQL.
3. Стандартная фиолетовая таблица конфигурации от `phpinfo()`.

Использование Docker Compose позволило полностью автоматизировать процесс настройки окружения, связать компоненты веб-сервера и базы данных в единую изолированную сеть, а также реализовать вывод динамических данных.
