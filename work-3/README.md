# Отчеты по практическим работам №3 и №4

Данный репозиторий содержит готовую инфраструктуру и код приложения для выполнения практических работ по развертыванию веб-серверов и созданию API.

---

# Отчет по Практической работе №3
**Тема:** Настройка серверов Apache и Nginx, проксирование и аутентификация через БД.

## 1. Цель работы
Создание сложной серверной конфигурации, сочетающей возможности Nginx (обработка статики, проксирование) и Apache (обработка динамического PHP-контента). Реализация системы аутентификации пользователей на основе данных из СУБД.

## 2. Архитектура решения
Проект развернут в Docker Compose и состоит из следующих контейнеров:
1.  **Nginx (Reverse Proxy):** Фронтенд-сервер. Слушает порт `8083`.
    *   Самостоятельно отдает статические файлы из папки `/app`.
    *   Проксирует PHP-запросы на бэкенд Apache.
2.  **Apache + PHP 8.2:** Бэкенд-сервер. 
    *   Root-директория изменена на `/app`.
    *   Включен `mod_rewrite` для обработки `.htaccess`.
    *   Установлены расширения `mysqli` и `pdo_mysql`.
3.  **MySQL 8.0:** База данных `restaurant_db`.

## 3. Описание реализации
*   **Проксирование:** Nginx настроен на передачу `.php` файлов через `proxy_pass http://apache:80`.
*   **Аутентификация:** Реализована Basic-авторизация в `src/auth.php`. Данные пользователей (логин/пароль) проверяются по таблице `users` в MySQL.
*   **Тестовое приложение:** Тема "Ресторан". Включает страницы меню (динамика) и информационные страницы (статика).

# Отчет по Практической работе №4
**Тема:** Реализация взаимодействия клиента и сервера с использованием технологии API.

## 1. Цель работы
Разработка REST API для управления данными ресторана (категории и блюда) с поддержкой всех CRUD-операций.

## 2. Спецификация API
*   **Эндпоинт:** `http://localhost:8083/api.php`
*   **Формат:** JSON (UTF-8).
*   **Сущности:** `categories`, `dishes`.
*   **Методы:**
    *   `GET`: Получение списка или записи по ID.
    *   `POST`: Создание новой записи.
    *   `PUT`: Обновление существующей записи.
    *   `DELETE`: Удаление записи.

## 3. Тестирование через Postman/Newman
Для автоматической проверки работоспособности всех функций API (CRUD) подготовлена коллекция тестов.

### Инструкция по запуску тестов:

1.  **Установка Newman** (если еще не установлен):
    ```bash
    npm install -g newman
    ```

2.  **Запуск контейнеров проекта**:
    ```bash
    docker compose up -d
    ```

3.  **Запуск тестов**:
    ```bash
    newman run restaurant_api_collection.json
    ```

### Что проверяют тесты:
*   **Доступность API:** Проверка ответа `200 OK`.
*   **Работа с категориями:** Создание новой категории, её получение по ID, переименование и последующее удаление.
*   **Работа с блюдами:** Добавление блюда в созданную категорию, проверка данных, обновление цены и удаление.
*   **Целостность JSON:** Проверка того, что сервер всегда возвращает валидный JSON-объект.

## 4. Вывод
Реализован полноценный REST API, интегрированный в инфраструктуру Nginx+Apache. Код защищен от SQL-инъекций (Prepared Statements) и настроен на корректную работу с кодировкой UTF-8. Автоматизированное тестирование подтверждает 100% работоспособность всех эндпоинтов.
