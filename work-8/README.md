# Отчет по Практической работе №8
**Тема:** Применение фреймворка Laravel для разработки веб-приложений.

## 1. Цель работы
Перенос функциональности, разработанной в рамках предыдущих работ (ПР №1–7), на современный PHP-фреймворк **Laravel**. Изучение архитектуры фреймворка, механизмов маршрутизации, использования ORM Eloquent, шаблонизатора Blade и встроенных инструментов безопасности.

## 2. Обоснование выбора фреймворка
Для реализации проекта был выбран **Laravel** по следующим критериям:
*   **Архитектура MVC:** Строгое разделение логики, данных и интерфейса «из коробки».
*   **Eloquent ORM:** Удобная работа с базой данных через объекты вместо написания SQL-запросов.
*   **Blade Templating:** Мощный шаблонизатор с поддержкой наследования и компонентов.
*   **Безопасность:** Встроенная защита от CSRF-атак, XSS и SQL-инъекций.
*   **Экосистема:** Наличие Artisan (CLI-интерфейс) и развитая система миграций и сидеров.

## 3. Описание реализации

### Архитектура (The Laravel Way)
1.  **Маршрутизация (`routes/web.php`):** Все процедурные обработчики заменены на именованные маршруты, которые обращаются к методам контроллеров.
2.  **Контроллеры (`app/Http/Controllers/`):** 
    *   `AdminController`: Управление добавлением блюд и запуском фикстур.
    *   `MenuController`: Логика отображения меню.
    *   `ChartController`: Генерация графиков (интеграция JpGraph в Laravel).
    *   `PreferenceController`: Управление сессиями и персонализацией через встроенный механизм сессий Laravel.
3.  **Модели и Eloquent (`app/Models/`):** 
    *   Созданы модели `Dish` и `Category` со связями `belongsTo` и `hasMany`.
    *   Вместо `mysqli` используется Eloquent, что делает код чище: `Dish::with('category')->get()`.
4.  **Шаблоны Blade (`resources/views/`):** 
    *   Использование директив `@if`, `@foreach`, `@extends` для динамического отображения контента.
    *   Реализована плавная поддержка тем (Dark/Light) и локализации через сессии Laravel.

### Сохранение функциональности
*   **Персонализация:** Тема и язык теперь хранятся в стандартных сессиях Laravel (вместо ручной работы с Cookies), что повышает безопасность.
*   **Фикстуры:** Перенесены в `database/seeders/RestaurantSeeder.php` и запускаются через `Artisan::call('db:seed')`.
*   **Графики:** Библиотека JpGraph интегрирована через Composer и выдает изображения через специальные маршруты `/charts/{type}`.
*   **Безопасность:** Доступ к админ-панели защищен посредником (middleware) `auth.basic`.

### Развитие REST API
API из предыдущих работ был полностью перенесен и модернизирован средствами Laravel:
*   **Контроллер:** Создан `Api\DishController` для обработки REST-запросов.
*   **Маршрутизация:** Используется `Route::apiResource`, что автоматически создает эндпоинты для GET, POST, PUT и DELETE.
*   **Валидация:** Все входящие данные проходят через встроенный валидатор Laravel перед сохранением в БД.
*   **JSON-ответы:** Реализованы корректные HTTP-коды ответов (201 Created, 422 Unprocessable Entity, 404 Not Found) и поддержка кириллицы.

## 4. Схема работы
1. Пользователь заходит на маршрут `/menu`.
2. `MenuController` извлекает данные через модель `Dish`.
3. Данные передаются в Blade-шаблон `resources/views/menu.blade.php`.
4. Шаблон рендерится с учетом настроек темы из сессии и возвращается пользователю.
5. Внешние системы могут взаимодействовать с приложением через `/api/dishes` для управления каталогом блюд.

## 5. Вывод
Перенос проекта на Laravel позволил значительно сократить объем шаблонного кода (boilerplate) и повысить надежность системы. Использование профессионального фреймворка позволило сосредоточиться на бизнес-логике приложения, доверив инфраструктурные задачи (безопасность, сессии, подключение к БД) проверенным инструментам Laravel. Проект стал полностью соответствовать современным стандартам веб-разработки.
