# Отчет по Практической работе №6
**Тема:** Сторонние модули PHP. Работа с библиотеками через Composer.

## 1. Цель работы
Изучение механизмов управления зависимостями в PHP с использованием **Composer**. Интеграция сторонних библиотек для генерации тестовых данных (фикстур) и построения графиков. Освоение работы с графической библиотекой GD для постобработки изображений.

## 2. Используемые инструменты и библиотеки
Для реализации функционала были выбраны следующие пакеты:
1.  **Faker (`fakerphp/faker`):** Библиотека для генерации реалистичных случайных данных (имена, числа, даты, тексты).
2.  **JpGraph (`amenadiel/jpgraph`):** Объектно-ориентированная библиотека для создания графиков и диаграмм.
3.  **Composer:** Инструмент для управления зависимостями, через который были установлены вышеуказанные библиотеки.
4.  **GD Library:** Системное расширение PHP для работы с изображениями, использованное для наложения водяных знаков.

## 3. Описание реализации

### Шаг 1. Генерация фикстур (`fixtures.php`)
С помощью библиотеки Faker был реализован скрипт для наполнения базы данных:
*   Сгенерировано **50 записей** для таблицы статистики.
*   Каждая запись содержит **5 полей**: название блюда (2 случайных слова), ID категории, цена, рейтинг (1-5) и дата создания.
*   Использована локализация `ru_RU` для генерации данных.

### Шаг 2. Построение графиков (`charts.php`)
На основе сгенерированных фикстур реализована страница статистики, отображающая 3 типа графиков:
1.  **Pie Plot (Круговая диаграмма):** Показывает процентное распределение рейтинга блюд в ресторане.
2.  **Line Plot (Линейный график):** Отображает динамику цен (на примере первых 20 позиций).
3.  **Bar Plot (Гистограмма):** Демонстрирует количество блюд в каждой категории.

### Шаг 3. Наложение водяного знака (GD)
Для защиты контента реализован механизм автоматического брендирования графиков:
*   Метод `Stroke(_IMG_HANDLER)` библиотеки JpGraph позволяет перехватить сгенерированное изображение.
*   Функция `imagettftext` накладывает текст "РЕСТОРАН" (или "RESTAURANT" в зависимости от выбранного языка).
*   Водяной знак имеет серый цвет и заданный уровень прозрачности (Alpha-канал), что соответствует требованиям задания.

## 4. Установка и настройка
Процесс установки сторонних модулей:
1.  Создание файла `composer.json` с перечислением необходимых библиотек.
2.  Запуск команды `composer install` внутри контейнера Apache (автоматизировано в `Dockerfile`).
3.  Подключение автозагрузчика `require_once 'vendor/autoload.php'` в PHP-скриптах.

## 5. Вывод
В ходе работы была успешно реализована интеграция стороннего кода в проект. Использование Composer позволило значительно ускорить разработку, делегировав сложные задачи (генерация данных и построение графиков) проверенным библиотекам. Реализованный функционал статистики с защитой изображений водяными знаками делает веб-приложение более информативным и профессиональным.
