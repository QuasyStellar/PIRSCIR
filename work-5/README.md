# Отчет по Практической работе №5
**Тема:** Сессии, Файлы и Базы Данных. Персонализация контента.

## 1. Цель работы
Модернизация веб-ресурса механизмами обработки сессий, персонализации пользовательского интерфейса (Content Negotiation) и системой управления файлами. Настройка внешнего хранилища сессий (Redis) и реализация различных способов хранения бинарных данных (PDF).

## 2. Новые компоненты инфраструктуры
К существующему стеку (Nginx + Apache + MySQL) был добавлен сервис **Redis**:
*   **Redis:** Используется как высокопроизводительное хранилище сессий в оперативной памяти. Это позволяет хранить состояние пользователя независимо от конкретного процесса Apache, обеспечивая лучшую масштабируемость.

## 3. Описание реализации

### Шаг 1. Хранение сессий в Redis
Настройка выполнена на уровне конфигурации PHP в контейнере Apache (`Dockerfile`):
*   Установлено расширение `php-redis` через PECL.
*   Директива `session.save_handler` установлена в значение `redis`.
*   Директива `session.save_path` указывает на адрес сервиса `tcp://redis:6379`.
Теперь при вызове `session_start()` данные сессии автоматически сохраняются в Redis.

### Шаг 2. Персонализация контента (Cookies)
Реализован механизм согласования контента на основе трех параметров, хранящихся в Cookie (файл `personalization.php`):
1.  **Логин пользователя:** Отображается в приветствии.
2.  **Тема оформления:** Светлая (по умолчанию) или Темная. Меняет цветовые схемы (`background`, `color`) на всех страницах.
3.  **Язык интерфейса:** RU или EN. Меняет текстовые метки (labels) во всем приложении.
Параметры применяются динамически во всех разделах сайта через подключение общего файла настроек.

### Шаг 3. Работа с файлами (PDF)
Реализована система загрузки и скачивания PDF-меню (`upload.php`, `download.php`):
*   **Хранение в ФС:** Файлы загружаются в защищенную директорию `uploads/` на сервере. В базе данных (`menu_files`) фиксируется только имя файла.
*   **Хранение в БД (BLOB):** Подготовлена структура в таблице `pdfs` для хранения файлов целиком в виде бинарных объектов (`MEDIUMBLOB`). Скрипт `download.php` демонстрирует отдачу файла напрямую из базы с правильным HTTP-заголовком `Content-Type: application/pdf`.
*   **Безопасность:** Реализована проверка расширения файла (разрешен только `.pdf`) и очистка путей.

### Шаг 4. Развитие REST API
API из работы №4 был расширен:
*   Добавлена поддержка методов **PUT** (обновление данных) и **DELETE** (удаление).
*   Теперь API поддерживает полный цикл CRUD (Create, Read, Update, Delete) для категорий и блюд.

## 4. Тестирование
1.  **Redis:** Проверено через `redis-cli monitor`, подтверждающий запись ключей сессий при входе.
2.  **Персонализация:** Смена темы и языка в настройках мгновенно отражается на всех страницах (Menu, Admin, Static).
3.  **Загрузка:** PDF-файлы успешно сохраняются в папку `uploads/` и отображаются в списке доступных для скачивания.

## 5. Вывод
В ходе работы была создана стабильная версия приложения с продвинутым управлением состоянием. Использование Redis для сессий повысило надежность, а механизмы персонализации и работы с файлами сделали систему более гибкой и удобной для конечного пользователя. Сохранена полная преемственность функционала предыдущих работ.
